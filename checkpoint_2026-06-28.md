## 一、等级公式

```js
等级 = floor(1 + str × 0.05 + con × 0.06 + dex × 0.04 + int × 0.02 + wis × 0.015 + luck × 0.015)
```

### 六维权重设计
| 属性 | 权重 | 设计理由 |
|------|------|----------|
| con(体质) | 6% | 最高权重，直接影响生命值和防御力，是怪物强度的核心指标 |
| str(力量) | 5% | 影响物理攻击力，第二权重 |
| dex(敏捷) | 4% | 影响速度和闪避，第三权重 |
| int(智力) | 2% | 影响魔法攻击和技能效果 |
| wis(精神) | 1.5% | 影响魔法防御 |
| luck(幸运) | 1.5% | 影响暴击和稀有掉落 |

### 升级所需经验
| 等级 | 所需经验 | 对应普通怪物数 | 说明 |
|------|----------|---------------|------|
| 1→2 | 10 | 5只 | 快速起步 |
| 2→3 | 20 | 10只 | 继续加速 |
| 3→4 | 30 | 15只 | 稳步提升 |
| 4→5 | 40 | 20只 | 临近毕业 |
| 5→6 | 50 | 25只 | 最后一站快轨 |
| 6→7 | 100 | 50只 | 进入慢阶段 |
| 7→8 | 110 | 55只 | 略增 |
| 8→9 | 120 | 60只 | 继续增长 |
| N→N+1(N≥6) | 100 + 10×(N-6) | 50 + 5×(N-6) | 线性增长 |

### 升级公式（代码实现）
```js
// 快阶段（1-5级）：每级所需经验 = 等级 × 10
// 慢阶段（6+级）：每级所需经验 = 100 + (等级 - 6) × 10
function getExpForLevel(level) {
    if (level <= 5) return level * 10;
    else return 100 + (level - 6) * 10;
}
```
