# 地牢模式重构 - 执行计划

## 阶段 1: DungeonMapGenerator - 地图生成器

### 目标
创建新的 `DungeonMapGenerator` 类，实现：
1. 35-40个节点随机生成（网格布局，横竖线不交叉）
2. 最短路径需要9场战斗到达Boss
3. 节点类型：start, combat(70%), event(30%), boss, reward
4. 迷雾机制：未访问显示"?"，已访问和相邻显示实际类型
5. 战斗节点完成后变为empty类型
6. 导出配置对象，方便后续调整数值

### 文件
- **新建**: `game-dev/src/world/dungeon-map-generator.js` - 地图生成器核心
- **修改**: `game-dev/src/world/dungeon-map-system.js` - 集成新生成器
- **修改**: `game-dev/src/world/scene-manager.js` - 支持新地牢类型
- **修改**: `game-dev/src/ui/expedition-system.js` - 添加新地牢选项

### 技术要点
- 使用分层网格布局（5-6层，每层5-8个节点）
- 非交叉连线规则（只允许水平向右 + 垂直相邻连接）
- BFS验证最短路径 >= 9个combat节点
- 迷雾系统与现有渲染集成
- 配置驱动设计

## 阶段 2: CombatRoomSystem - 战斗场地

### 目标
实现战斗场地系统：
1. 随机生成1024-2048大小的正方形场地
2. 玩家随机生成在四边中间位置
3. 怪物生成在对边位置
4. 战斗完成后删除场地（恢复原始地形）
5. 与现有WallSystem和Camera系统兼容

## 阶段 3: EventSystem - 随机事件

### 目标
实现5种随机事件：
1. 女神像：恢复/祝福(+15%攻击,3场)/奖励
2. 陷阱：解除(敏捷)/跨越(体质)，失败扣25%血
3. 补给堆：搜寻(精神)/探查(敏捷)
4. 宝箱：50%金币/25%材料/25%战斗
5. 恶魔雕像：扣50%血魔，+33%攻击或材料

属性检定公式：成功率 = 基础 + 属性×1%，上限95%
Buff系统：阅读现有buff代码，实现女神祝福和恶魔祈祷
