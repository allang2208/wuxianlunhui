```
│   └── weaponParams.scale     // 缩放比例
│
├── 图片缓存
│   ├── charImage          // 角色待机图
│   ├── _charFrames        // 角色动画帧（idle/walk/running）

├── 绘制系统
│   ├── _draw()            // 主绘制：网格→坐标轴→角色→武器
│   ├── _getCharacterImage() // 根据动画状态获取角色图
│   └── _loadCharacterFrames() // 加载角色动画帧
│
├── 交互系统
│   ├── _onMouseDown()     // 左键放置/拖拽武器
│   ├── _onMouseMove()     // 拖拽中更新武器位置
│   ├── _onWheel()         // 滚轮缩放/旋转（R键切换模式）
│   └── 拖放：从右侧武器预览拖到 Canvas
│
├── 保存系统
│   ├── _save()            // 将屏幕偏移 → WeaponAnimConfig.holdOffsetX/Y
│   └── _reset()           // 重置为默认值
│
└── 坐标工具
    └── _startCoordTool()  // 全屏坐标测量工具（右键退出）
```

### 3. 武器→游戏配置的转换链

开发工具中的**屏幕偏移**（以角色中心为原点，Y+向上）需要通过复杂的数学变换，转换为游戏中 `WeaponAnimConfig` 的 **holdOffset**（以武器旋转中心为原点的局部坐标）。

```
开发工具坐标系（屏幕空间）          游戏内坐标系（局部旋转空间）
     Y+ ↑                                ↑ Y+ (after rotation)
        │   offsetX, offsetY              │
        │   (用户拖拽的直观位置)          │  holdOffsetX/Y
        │                                 │  (武器旋转中心)
   ─────┼─────→ X+                    ───┼───→ X+
        │ 角色中心                        │
                                         │  mainBaseX/Y
                                         │  (武器类型基准偏移)
```

**转换公式（按武器类型不同）：**

| 武器类型 | 游戏变换链 | 转换公式 |
|----------|-----------|----------|
| **近战** | `translate(mainBaseX, mainBaseY)` → `translate(holdOffsetX, holdOffsetY)` → `rotate(π/2)` → `translate(0, -ms*0.85)` | `holdOffsetX = offsetX - ms*0.85 + 7` |
| **枪械** | `translate(mainBaseX, mainBaseY)` → `translate(holdOffsetX, holdOffsetY)` → `rotate(π/2)` → `translate(0, -s*0.42)` | `holdOffsetX = offsetX - 104.6` |
| **弓** | `translate(mainBaseX, mainBaseY)` → `translate(holdOffsetX, holdOffsetY)` → `rotate(π/2)` | `holdOffsetX = offsetX + 7` |

> **关键发现**：这个转换链是**单向且不可逆**的——开发工具只能调整 `holdOffsetX/Y` 和 `idleRotation`，无法调整 `mainBaseX/Y`（游戏中武器类型的基准偏移）。这意味着即使你在开发工具中把武器拖到角色右手，保存后游戏中武器位置也可能因为 `mainBaseX/Y` 的固定值而不在理想位置。

## 三、挂载点开发可行性评估

### 3.1 当前开发工具能做什么 ✅

| 功能 | 实现状态 | 用途 |
|------|----------|------|
| 拖拽放置武器到画布 | ✅ 完成 | 直观调整武器位置 |
| 滚轮缩放武器 | ✅ 完成 | 调整武器大小匹配角色 |
| 滚轮旋转武器（R键切换模式） | ✅ 完成 | 调整武器角度 |
| 实时预览武器与角色的相对位置 | ✅ 完成 | 验证挂载效果 |
| 保存到 `WeaponAnimConfig` | ✅ 完成 | 自动同步到游戏 |
| 坐标工具 | ✅ 完成 | 测量屏幕坐标 |
| 切换动画状态（idle/walk/running） | ✅ 完成 | 查看不同状态下的武器位置 |
| 切换武器类型 | ✅ 完成 | 为不同武器独立设置参数 |

### 3.2 当前开发工具不能做什么 ❌

| 缺失功能 | 影响 | 严重程度 |
|----------|------|----------|
| **动画循环播放** | 只能看到静态帧，无法观察武器在动画播放时的动态效果 | 🔴 高 |
| **按帧设置挂载点** | walk 21帧、running 16帧中每帧角色手的姿态不同，固定挂载点会导致动画播放时武器与手错位 | 🔴 高 |
| **按动画状态独立设置** | 游戏内 `WeaponAnimConfig` 只有一套参数，无法为 idle/walk/running 分别设置 | 🟡 中 |
| **挂载点与角色骨骼绑定** | 无法定义"右手"骨骼点并让武器自动跟随 | 🟡 中 |
| **mainBaseX/Y 调整** | 游戏中 `mainBaseX/Y` 是硬编码的（近战-7,0 / 手枪-15,16.5 / 枪械8,0），开发工具无法调整 | 🟡 中 |
| **多方向视角** | 只能看到角色正面（或旋转后的正面），无法看到角色朝向左右时的武器位置 | 🟢 低 |
| **武器翻转同步** | 开发工具不处理角色面朝左右时的武器水平翻转 | 🟢 低 |

### 3.3 核心矛盾分析

**开发工具的设计假设** vs **精灵图动画的现实**：

```
开发工具假设：                    精灵图现实：
┌─────────────────────┐          ┌─────────────────────┐
│ 角色是一个静态图形   │          │ 角色有 21/16 帧动画  │
│ 武器挂载点固定不变   │          │ 每帧手的位置都不同   │
│ 一套参数适用于所有帧 │          │ 需要按帧微调挂载点   │
└─────────────────────┘          └─────────────────────┘
```

**结论**：如果精灵图是静态图（如 RPG Maker 的 4方向行走图），固定挂载点可以工作。但你的 walk/running 精灵图有 21/16 帧，角色手臂在动画过程中会摆动，**固定挂载点会导致武器在动画播放时与手错位**。

## 四、改进方案（按优先级排序）

### 方案 A：最小可行改进（推荐先做）

**目标**：让开发工具能用于"大致对齐"右手，适合快速迭代

**改动**：

1. **添加帧导航滑块**
   - 在动画选择下方添加滑动条："帧: [1] ─────── [21]"
   - 可以切换到 walk/running 的任意帧
   - 当前显示帧编号

2. **每帧显示一个参考点**
   - 在角色图右手位置画一个半透明圆点作为参考
   - 用户把武器中心对齐到这个圆点即可

3. **添加动画预览按钮**
   - 点击"播放"按钮，walk/running 在画布上循环播放
   - 实时观察武器是否跟随手移动
   - 点击"暂停"可以定格到任意帧进行调整

4. **调整 sprite 绘制尺寸**
   - 当前开发工具绘制 120×120，但游戏内实际绘制 `size * 4.5` ≈ 72px（或 `size * 6.25` ≈ 100px，视上下文）
   - 需要统一尺寸，避免"开发工具看起来对，游戏里不对"

**工作量**：约 2-3 小时
**效果**：可以精确对齐到单帧，但所有帧共享同一套挂载点参数

---

### 方案 B：按帧独立挂载点（推荐中期实现）

**目标**：为 walk 的 21帧和 running 的 16帧分别设置挂载点

**改动**：

1. **扩展数据模型**
   ```javascript
   // 当前 WeaponAnimConfig（单套参数）
   sword: { holdOffsetX: -35, holdOffsetY: 4, ... }
   
   // 扩展后（按动画状态 + 按帧）
   sword: {
     idle: { holdOffsetX: -35, holdOffsetY: 4, ... },
     walk: {
       frames: [
         { holdOffsetX: -35, holdOffsetY: 4 },   // 第1帧
         { holdOffsetX: -36, holdOffsetY: 3 },   // 第2帧
         // ... 21帧
       ]
     },
     running: { frames: [...] }  // 16帧
   }
   ```

2. **开发工具支持帧级编辑**
   - 选中 walk 动画后，显示 21 个帧缩略图
   - 点击缩略图切换到该帧
   - 在该帧上独立调整武器位置
   - 提供"应用到所有帧"按钮（批量设置）
   - 提供"插值"功能：设置第1帧和第21帧，自动计算中间帧

3. **游戏中按帧读取挂载点**
   - 修改 `player.js` 的 `renderWeapon()`
   - 根据当前动画帧索引读取对应的挂载点

**工作量**：约 1-2 天
**效果**：武器在所有动画帧中都能精确对齐到右手
**风险**：数据量增大（21+16=37帧 × 2个坐标 = 74个数值），手工调整繁琐

---

### 方案 C：骨骼绑定系统（推荐长期实现）

**目标**：不依赖逐帧数据，而是定义角色骨骼点，让武器自动跟随

**改动**：

1. **定义角色骨骼点**
   - 为每个动画帧标注关键骨骼点：右手 `(handR)`、左手 `(handL)`、头部 `(head)`
   - 每个骨骼点标注在 sprite sheet 中的 `(x, y)` 坐标
   - 使用 JSON 文件存储：`character-skeleton.json`

   ```json
   {
     "idle": { "handR": { "x": 320, "y": 180 } },
     "walk": {
       "frames": [
         { "handR": { "x": 318, "y": 178 } },
         { "handR": { "x": 322, "y": 182 } },
         // ...
       ]
     }
   }
   ```

2. **开发工具添加骨骼标注模式**
   - 切换到"骨骼标注"模式
   - 在 Sprite Sheet 上点击标注骨骼点位置
   - 自动生成 JSON

3. **游戏中武器挂载到骨骼**
   - 武器不再使用 `holdOffsetX/Y`
   - 而是使用 `attachTo: 'handR'`
   - 渲染时查询当前帧的 `handR` 位置，武器自动跟随

4. **支持多武器类型**
   - 单手武器 → 挂载到 `handR`
   - 双手武器 → 挂载到 `handR` + `handL`（两点对齐）
   - 盾牌 → 挂载到 `handL`

**工作量**：约 3-5 天
**效果**：最灵活、最自然的挂载系统，后续添加新武器/新动画无需重新调整挂载点
**风险**：需要为所有精灵图的所有帧手工标注骨骼点，初始工作量大

---

### 方案 D：快速取巧方案（推荐临时方案）

**目标**：不修改游戏代码，仅调整现有参数，让武器"看起来"在右手

**做法**：

1. 在开发工具中，选择 `walk` 动画，找到**中间帧**（第10帧左右）
2. 把武器拖到右手位置，记录 offsetX/Y
3. 点击保存，应用到 `WeaponAnimConfig`
4. 在开发工具中，点击"播放"观察动画循环
5. 如果某些帧错位严重，微调 `mainBaseX/Y`（需要修改 `player.js` 中的硬编码值）

**关键技巧**：
- 由于 `mainBaseX` 在 `player.js` 中是硬编码的（近战-7, 手枪-15, 枪械8），如果开发工具中看起来对但游戏中不对，需要同时修改 `player.js` 中的 `mainBaseX/Y` 和开发工具中的转换公式。
- 建议：把 `mainBaseX/Y` 也加入 `WeaponAnimConfig`，让开发工具可以一并调整。

**工作量**：约 30 分钟
**效果**：能用，但动画播放时会有轻微错位

## 六、具体操作建议（现在就可以做）

1. **打开游戏，按 `T` 键打开开发工具**
2. **选择 "行走" 动画，找到手臂位置最清晰的帧**
3. **从右侧拖拽一把武器到角色右手**
4. **微调位置，让武器中心对齐到右手**
5. **按 `R` 键切换旋转模式，滚轮调整武器角度**
6. **点击"保存"，自动应用到游戏**
7. **关闭开发工具，在游戏中按 Shift 奔跑/不按 Shift 行走，观察武器是否跟随手**
8. **如果错位，重复步骤 2-7**

**注意**：由于当前游戏代码中 `mainBaseX/Y` 是硬编码的，开发工具中的转换公式可能不完全匹配。如果发现保存后游戏中位置不对，需要检查 `player.js` 中对应武器类型的 `mainBaseX/Y` 值，与开发工具 `_save()` 中的转换公式是否一致。

## 七、已知问题与风险

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| 转换公式不匹配 | 开发工具 `_save()` 中的 `holdOffsetX` 转换公式可能与 `player.js` 中 `renderWeapon()` 的变换链不完全一致 | 需要逐一验证每种武器类型的转换公式 |
| Sprite 绘制尺寸不一致 | 开发工具绘制 120×120，但游戏内实际绘制 72px（或 100px） | 统一为游戏内实际尺寸 |
| 武器翻转未处理 | 角色面朝左时武器需要 `scale(-1, 1)` 翻转，开发工具未展示 | 添加方向切换按钮 |
| 呼吸抖动未同步 | idle 状态的呼吸抖动（sway/scale）可能影响武器视觉位置 | 在开发工具中同步绘制呼吸抖动 |

---

**总结**：当前开发工具**可以用于挂载点的基础对齐**，但要做精确的"每帧手对齐"需要**方案 A（帧导航+动画预览）**或**方案 B（按帧独立挂载点）**。建议先用方案 D 快速验证效果，再决定是否投入时间做更精细的方案。
