# 侧视角 2D 迁移技术评估

> 日期：2026-07-04
> 评估者：Kimi Agent
> 状态：评估完成，未实施

## 一、澄清目标

用户意图：**保留挺进地牢核心机制**（地牢房间、翻滚、射击、无重力）+ **视觉呈现改为侧视角**。

挺进地牢本身是**俯视角**（Top-Down），而非侧视角。用户想要的应该是类似 **Hades** 或 **Dead Cells** 的表现方式：角色从侧面看，但移动仍是 2D 平面（上下左右自由移动），没有重力/跳跃。

```
当前：俯视角（看头顶）          目标：侧视角（看侧面轮廓）
     ↑                              ↑
     │                              │
  ┌──┴──┐                       ┌──┴──┐
  │  角色  │                       │ 角色 │  ← 侧视精灵
  │ (圆)  │                       │ 侧面 │
  └──┬──┘                       └──┬──┘
     │                              │
```

## 二、核心机制不变（确认）

| 机制 | 当前 | 目标 | 是否需要修改 |
|------|------|------|-----------|
| 坐标系 | 纯2D `(x, y)` | 纯2D `(x, y)` | ❌ 否 |
| 重力 | 无 | 无 | ❌ 否 |
| 跳跃 | 无 | 无 | ❌ 否 |
| 碰撞 | 圆形↔矩形 | 圆形↔矩形 | ❌ 否 |
| 寻路 | A* 2D 网格 | A* 2D 网格 | ❌ 否 |
| 射击 | 任意角度 | 任意角度 | ❌ 否 |
| 翻滚/冲刺 | 2D 方向 | 2D 方向 | ❌ 否 |

**结论：底层逻辑几乎完全不需要改动。**

## 四、关键技术点

### 1. 深度排序（Renderer）

侧视角下，y 坐标代表"远近"，需要确保远的实体先渲染，近的实体后渲染（遮挡关系）：

```javascript
// Renderer.renderEntities() 中新增
const sortedEntities = Array.from(entities.values())
    .sort((a, b) => a.y - b.y); // y 小的（远的）先画
```

### 2. 朝向翻转（Player/Enemy）

```javascript
render(ctx) {
    const screenPos = Renderer.worldToScreen(this.x, this.y);
    const facingRight = this.vx >= 0; // 或根据鼠标位置判断
    
    ctx.save();
    ctx.translate(screenPos.x, screenPos.y);
    if (!facingRight) ctx.scale(-1, 1); // 向左时水平翻转
    
    // 绘制侧视角贴图
    ctx.drawImage(this.sideViewImage, -w/2, -h/2, w, h);
    
    ctx.restore();
}
```

### 3. 武器角度限制

当前武器可以朝任意角度。侧视角下，武器只需要左右两个方向：

### 4. 墙壁渲染

从俯视的顶面渲染改为侧视的立面渲染：

## 五、素材需求

| 类型 | 当前素材 | 需要替换为 |
|------|---------|-----------|
| 玩家角色 | 俯视 8 方向行走图 | 侧视 2 方向（左/右）行走图 |
| 敌人 | 俯视精灵 | 侧视精灵 |
| 武器（装备栏） | 俯视角度 | 侧视角度 |
| 墙壁 | 俯视顶面 | 侧视立面（带厚度） |
| 树木/障碍物 | 俯视圆形 | 侧视轮廓 |

## 六、实施建议

### 最小可行方案（MVP）

1. **Phase 1：玩家角色侧视角化**
   - 创建侧视角玩家贴图（左/右两个方向）
   - 修改 `Player.render()`：根据移动方向翻转，绘制侧视贴图
   - 保留所有移动/射击逻辑不变
   - **预期工作量**：1-2 天

2. **Phase 2：敌人和环境**
   - 敌人改为侧视角贴图
   - 墙壁/地面改为侧视渲染
   - 添加深度排序
   - **预期工作量**：2-3 天

3. **Phase 3：特效和动画**
   - 攻击动画、受击特效、子弹轨迹适配侧视角
   - **预期工作量**：1-2 天

### 关键决策

| 决策 | 选项 A（推荐） | 选项 B |
|------|---------------|--------|
| 鼠标瞄准 | 保留 360°，子弹任意角度 | 限制为水平 ±45° |
| 上下移动动画 | 播放左右动画（y 移动不翻转） | 需要上下行走的动画帧 |
| 武器朝向 | 跟随鼠标角度（任意旋转） | 锁定为左/右水平 |
| 翻滚方向 | 保留 8 方向翻滚 | 限制为 2 方向 |

**推荐选项 A**：保持核心玩法不变（挺进地牢式），只改变视觉呈现。

## 七、风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 侧视角贴图资源不足 | 高 | 高 | 使用 AI 生成或购买素材包 |
| 上下移动时视觉怪异 | 中 | 中 | 添加轻微的上下浮动动画 |
| 深度排序性能问题 | 低 | 中 | 每帧排序改为有变化时才排序 |
| 与现有特效不兼容 | 中 | 低 | 逐步替换特效 |

## 八、总结

- **底层逻辑改动量**：< 5%（几乎不变）
- **渲染层改动量**：~ 30%（角色、敌人、墙壁、特效）
- **核心工作量**：素材替换（侧视角贴图）> 代码改动
- **建议方案**：先做玩家角色的侧视角原型验证，确认手感后再全面迁移

---

> **注意**：挺进地牢本身是俯视角游戏。如果用户想要的是类似 Dead Cells（纯侧视平台）或 Hades（等距俯视角但角色有侧面感），需求会有不同。建议在实施前确认具体的视觉参考。
